# 数据库配置问题解决指南

## 错误信息
`Access denied for user 'root'@'localhost' (using password: YES)`

这个错误表示数据库密码不正确或用户权限不足。

## 解决方法

### 方法1：修改配置文件中的密码

编辑文件：`src/main/resources/config/database.properties`

修改以下内容：
```properties
db.url=jdbc:mysql://localhost:3306/bigcomp_access?useSSL=false&serverTimezone=UTC&characterEncoding=utf8
db.username=root
db.password=你的实际MySQL密码  # 改为你的MySQL root密码
```

### 方法2：检查MySQL服务

1. **检查MySQL是否运行**：
   - Windows: 打开"服务"管理器，查找MySQL服务，确保状态为"正在运行"
   - 或在命令行运行：`net start MySQL`（如果服务名为MySQL）

2. **测试数据库连接**：
   在命令行运行：
   ```bash
   mysql -u root -p
   ```
   输入密码，如果能连接成功，说明MySQL正常。

### 方法3：重置MySQL密码（如果忘记密码）

如果忘记了MySQL root密码，可以：

1. **停止MySQL服务**
2. **以跳过权限表方式启动MySQL**（具体步骤取决于MySQL版本）
3. **重置密码**

### 方法4：创建新的数据库用户（推荐）

为了安全，建议创建一个专门的数据库用户：

```sql
-- 在MySQL中执行
CREATE USER 'access_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON bigcomp_access.* TO 'access_user'@'localhost';
FLUSH PRIVILEGES;
```

然后在配置文件中使用：
```properties
db.username=access_user
db.password=your_password
```

## 快速测试

修改配置文件后，重新打包并运行：
```bash
mvn package
java -jar target\access-control-system-1.0.0.jar
```
